# Homepage I18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the generic homepage visual and add a first-pass Chinese/English internationalized VitePress structure.

**Architecture:** Chinese remains the default VitePress site at `/`, while English pages live under `/en/` through VitePress built-in locale support. The homepage image is a repository asset under `docs/public/images/`, and README language switching is handled with Markdown anchors in a single file.

**Tech Stack:** VitePress 1.x, Markdown frontmatter, TypeScript site config, static SVG/PNG assets, npm build scripts.

### Task 2: VitePress Locale Configuration

**Files:**
- Modify: `docs/.vitepress/config.mts`

- [ ] **Step 1: Keep Chinese as default**

Use the existing Chinese nav/sidebar as the root locale config and add root locale labels:

```ts
lang: 'zh-CN'
```

- [ ] **Step 2: Add English locale**

Add a `locales.en` config with:

- `label: 'English'`
- `lang: 'en-US'`
- `link: '/en/'`
- English nav entries for Home, About, Support, Guide, GitHub
- English sidebar for `/en/about/`
- English theme labels for local search and last updated

- [ ] **Step 3: Preserve existing routes**

Do not rename or move existing Chinese Markdown files.
